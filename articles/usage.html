Usage

Yes, as you might guess this website uses GitHub Pages Router. You can also get started taking a look to its index.html as a working example.

Installation

First of all import somehow github-pages-router.js in your HTML page. Remember that you need both index.html and 404.html.

There is a github-pages-router package on npm so for example you can use a CDN like 

<script src="https://unpkg.com/github-pages-router"></script>

Declare your routes

Add a single ghp-router tag in your HTML, and one ghp-route tag for every of your routes. Something like 

<ghp-router>

  <ghp-route
    href="./route1"
    content="./path/to/content1.html"
  ></ghp-route>

  <ghp-route
    href="./route2"
    content="./path/to/content2.html"
  ></ghp-route>

  <!-- Your HTML layout -->

  <main>
    <!--
      Leave this empty.
      Your route related HTML content will be loaded here.
    -->
  </main>

</ghp-router>

The ghp-router element accepts an optional outlet attribute: it must be a selector that matches the element that wraps the content; its value defaults to main.

Internal links

Use a ghp-link or a ghp-navlink tag to wrap your internal anchors. For example: 

<ghp-link>
  <a href="./about">About</a>
</ghp-link>

The ghp-navlink will add an aria-current="page" ARIA attribute to the anchor it wraps whenever it points to the current page. It can be used to highlight current selected link, for example in this website I added the following CSS 

nav a[aria-current="page"] {
  background-color: hsla(200, 60%, 60%, 0.2);
  cursor: default;
}
